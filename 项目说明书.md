# YuhanBakery (小龙虾烘焙坊) 项目说明书

## 一、 为什么做项目 & 项目背景

随着新零售和 O2O（Online To Offline）模式的普及，传统的线下烘焙门店面临着数字化转型的需求。为了拓宽销售渠道、提升用户体验并优化门店的配送和运营效率，**YuhanBakery（小龙虾烘焙坊）** 应运而生。

传统烘焙店在日常经营中通常面临以下痛点：
1. **获客渠道单一：** 仅依赖线下客流，难以触达更广泛的消费者。
2. **点单与排队效率低：** 高峰期顾客排队时间长，体验不佳。
3. **配送体系不完善：** 缺乏一个高效的自建或整合配送的系统，骑手接单与门店发货脱节。
4. **营销手段匮乏：** 缺乏优惠券、满减等灵活的数字化营销工具。

**项目背景：**
YuhanBakery 旨在通过打造一个全链路的线上商城系统，打通“顾客下单 - 门店接单 - 骑手配送”的业务闭环。通过前后端分离的现代化架构，为顾客提供流畅的购物体验，为门店管理员提供便捷的后台管理能力，并为外卖骑手提供清晰的接单派送大厅。

---

## 二、 项目整体介绍

YuhanBakery 是一个基于 **Java 21** 和 **Spring Boot 3** 构建的现代化烘焙坊线上商城系统。它涵盖了完整的电商业务链路，不仅支持基础的商品展示和购物车，还深度集成了营销玩法（优惠券体系）和即时配送业务。

项目分为三个核心角色和对应的业务模块：
- **顾客端：** 聚焦于购物体验（浏览、加购、领券、支付、评价）。
- **管理端：** 聚焦于门店运营（商品上下架、订单发货、营销配置、数据统计）。
- **骑手端：** 聚焦于末端物流（订单抢单、取货、送达状态更新）。

## 三、 功能说明

系统根据不同的角色，划分了以下核心功能模块：

### 1. 顾客端功能 (Customer)
- **商品与分类浏览：** 按分类查看各类面包、蛋糕、饮品等，支持查看商品详情。
- **购物车与智能结算：** 灵活加购商品；在结算台可**动态拉取可用优惠券**，自动校验门槛并计算抵扣后金额，下单后平滑跳转至订单跟踪页。
- **🤖 AI 智能客服助理：** 集成火山引擎大模型（DeepSeek），通过 SSE（Server-Sent Events）实现流式打字机交互，沉浸式解答商品疑问和提供营销导购。
- **订单与支付：** 提交订单，支持模拟微信支付，实时查看订单状态（待支付、待发货、配送中、已完成）。
- **用户中心：** 个人信息管理，收货地址增删改查，订单历史与商品评价。

### 2. 管理端功能 (Admin)
- **商品与分类管理：** 灵活配置商品类目，上架/下架商品，配置商品规格、价格及展示图片（基于 OSS）。
- **套餐管理 (Combo)：** 将多个商品组合为套餐售卖，提升客单价。
- **订单处理：** 查看全店订单，处理用户退款，调度发货。
- **营销管理：** 配置和发放优惠券（满减、折扣等），管理发放数量和有效期。
- **数据与资产统计：** 统计每日订单量、营业额、用户余额变更日志等，为运营提供数据支撑。

### 3. 骑手端功能 (Rider)
- **极致的沉浸式工作台：** 采用赛博朋克/机能风（Neo-Brutalism）暗黑设计，配备高对比度自发光组件，提供极佳的户外可读性。
- **滑动确认与流转机制：** 采用防误触的**两段式滑动按钮（Double Swipe）**进行取餐和送达确认，彻底告别误点。
- **真实资金流水系统：** 骑手完成配送后运费即时入账，支持发起真实“提现（Withdraw）”，底层直接扣减 DB 账户余额并生成 `user_balance_log`。

## 四、 技术方案

为了支撑高并发的电商场景和良好的扩展性，YuhanBakery 采用了业界主流的微服务/单体架构技术栈，并引入了多种中间件。

### 1. 核心技术栈
- **开发语言：** Java 21
- **核心框架：** Spring Boot 3.5.x
- **安全认证：** Spring Security + JWT（实现无状态的 RBAC 角色权限控制）
- **持久层框架：** MyBatis-Plus 3.5.x
- **数据库：** MySQL 8.x

### 2. 中间件与云服务
- **缓存与高速读写：** **Redis**
  - 用于缓存高频访问的数据（如商品分类列表）。
  - 存储用户的 Token 信息，实现单点登录和 Token 吊销机制。
  - 作为分布式锁的底层实现，防止商品超卖和优惠券重复领取。
- **消息队列：** **RabbitMQ**
  - 削峰填谷：应对高并发下单场景。
  - 异步解耦：用于订单超时未支付自动取消（延迟队列机制）、支付成功后的积分发放和通知推送。
- **对象存储：** **阿里云 OSS**
  - 集中管理商品图片、用户头像等静态资源，减轻应用服务器带宽压力。

### 3. 系统架构设计亮点
- **前后端分离：** 后端提供标准的 RESTful API，配合 SpringDoc OpenAPI（Swagger 3）自动生成接口文档，方便前端及移动端对接。
- **全局异常与参数校验：** 统一的 `Result` 响应体和全局异常处理器（GlobalExceptionHandler），结合 Spring Validation，保证接口的健壮性。
- **自动化的数据初始化：** 结合 Spring Boot SQL 初始化脚本（`schema.sql` 和 `data.sql`），支持一键启动和环境迁移。
- **Docker 容器化：** 提供 `Dockerfile` 和 `docker-compose.yml`，实现 MySQL、Redis、RabbitMQ 及后端服务的秒级一键部署。
