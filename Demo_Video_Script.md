# YuhanBakery (小龙虾烘焙坊) - 项目展示视频剧本

**视频预计时长：** 3-5 分钟
**演示逻辑：** 以一个完整的“用户下单 -> 后台接单发货 -> 骑手接单送达 -> 用户评价”的 O2O 闭环为主线，串联起各端的核心功能。
**准备工作：** 提前在本地或服务器运行好项目后端服务，准备好前端页面或 API 测试工具（如 Postman/Swagger）。如果包含前端 UI，请分别打开三个不同的浏览器窗口或无痕模式代表三端。

---

## 🎬 开场白 (0:00 - 0:30)

**动作：** 
展示项目的 GitHub 首页或 Swagger UI 接口文档页面，随后切到系统的登录页或首页。
**台词：**
> “大家好，欢迎来到我的实战项目演示。今天给大家展示的是我自己开发的**YuhanBakery（小龙虾烘焙坊）**线上商城系统。
> 
> 这个项目是基于最流行的 Spring Boot 3 和 Java 21 开发的，采用前后端分离架构，核心解决的是烘焙门店从线上拓客到线下配送的 O2O 业务痛点。
> 
> 整个系统分为三个端：**顾客端**、**管理端**和**骑手端**。接下来，我将模拟一次完整的真实购物流程，带大家体验各个端的核心功能。”

---

## 🛍️ 第一幕：顾客端 - 选购与下单 (0:30 - 1:30)

**动作：** 
1. 顾客视角登录系统。
2. 浏览商品列表（随机点击几个分类，展示图片和价格）。
3. 演示“领取优惠券”功能。
4. 将两个商品加入购物车，然后打开购物车。
5. 点击结算，选择刚刚领取的优惠券，提交订单。
6. 展示“模拟微信支付”成功，订单状态变更为“待发货”。

**台词：**
> “首先，我们来到**顾客端**。假设我是一位来买早餐的顾客。
> 
> 大家可以看到，首页展示了各种分类的面包和蛋糕，这些图片都是通过阿里云 OSS 进行存储和加速的。在下单前，我们先去领券中心，领取一张‘满 50 减 10’的优惠券，这里的领券逻辑底层用了 Redis 分布式锁，防止了超发和重复领取。
> 
> 接着，我挑两个可颂加入购物车。点击结算，系统自动识别了我们可以使用的优惠券。提交订单后，我们调用了模拟的微信支付接口。支付成功！现在这笔订单的状态已经变成了‘待发货’。”

---

## 💻 第二幕：管理端 - 接单与调度 (1:30 - 2:30)

**动作：**
1. 切换到管理端的浏览器窗口。
2. 展示“订单管理”列表，刚刚顾客下的单高亮显示在最前面。
3. 点击“发货/分配骑手”操作。
4. （可选扩展）快速切到“商品管理”和“营销管理”，简单展示一下后台配置商品和发放优惠券的界面。

**台词：**
> “订单支付成功后，我们现在切换到店长的视角，也就是我们的**管理端**。
> 
> 在‘订单管理’列表中，我们立刻看到了刚才那笔新订单。如果是高峰期，系统内部使用了 RabbitMQ 消息队列来进行削峰填谷，保证订单不会丢失。
> 
> 店长核对好商品后，点击‘发货’，系统会将这笔订单推送到周边的接单大厅。
> 
> 顺便提一下，在这个强大的管理后台，店长不仅可以处理订单，还可以灵活地上架新商品、配置特价套餐，甚至自定义发放各种营销优惠券。所有的权限控制都由 Spring Security 和 JWT Token 来安全把关。”

---

## 🛵 第三幕：骑手端 - 抢单与配送 (2:30 - 3:30)

**动作：**
1. 切换到骑手端的浏览器窗口（或移动端模拟器）。
2. 在“接单大厅”刷新，看到刚刚发出的订单。
3. 点击“接单”。
4. 点击“已到店取货”（状态变更）。
5. 点击“确认送达”（状态变更）。

**台词：**
> “现在，订单已经到了我们的**骑手端**大厅。
> 
> 骑手小哥打开系统，在待接单列表中看到了这笔订单，立刻点击抢单。抢单成功后，骑手前往门店取货，点击‘已取货’，然后开始配送。
> 
> 抵达顾客地址后，骑手点击‘确认送达’。整个物流状态在后端是实时流转的，并且每一次状态更新，系统都有严密的逻辑校验。”

---

## ⭐ 第四幕：闭环与数据统计 (3:30 - 4:30)

**动作：**
1. 切回**顾客端**，刷新订单列表，展示订单已完成。顾客点击“评价”，打 5 星并随便写两句好评。
2. 切回**管理端**的“数据看板（统计页面）”，展示今日订单量 +1，今日营业额增加了刚刚的那笔金额。

**台词：**
> “最后，我们切回到**顾客端**。可以看到刚才的订单状态已经变成了‘已完成’。顾客吃得非常满意，顺手留下了一个全五星好评，一次完美的购物体验就此闭环。
> 
> 我们再回到**管理端**的数据看板。由于刚才的这笔订单，今日的订单总量和营业额都实时发生了增长，管理层可以清晰地看到门店的运营健康度。”

---

页面停留在系统的首页或是架构图/代码结构页，主播露脸或给出项目的 GitHub 二维码/链接。

**台词：**
> “以上就是 YuhanBakery 的全流程演示。
> 
> 这个项目不仅有完善的业务逻辑，底层还融入了高并发处理机制、分布式缓存、消息队列等企业级开发技术。如果您对源码感兴趣，或者想了解更多技术细节，欢迎访问我的 GitHub 获取完整代码并给我点个 Star！
> 
> 感谢大家的观看，我们下次再见！”
